# Contributing to the Open Labor Foundation

> **A note on contributions:** This repository is public for transparency, but is not yet accepting external issues or pull requests directly. Issues are disabled repo-wide, and pull requests from outside collaborators aren't reviewed at this stage. This is expected to change as the project matures — check back, or watch [openlabor.foundation](https://openlabor.foundation) for updates.

Thank you for your interest in contributing.

## The most valuable contributions

The foundation grows through people who know their field. If you have expertise in any profession, trade, or industry — and can describe what an AI specialist in that field should do, refuse, and know — that knowledge belongs in the catalog.

No engineering background required.

## How to contribute a specialist definition

1. Open an issue in [olf-agents](https://github.com/Open-Labor-Foundation/olf-agents) describing your domain
2. A maintainer will work with you to format it as a structured spec
3. Submit a pull request following the spec format in that repository

## Code contributions

For software contributions to any repository in this organization:

- Open an issue before starting significant work
- Follow the contributing guide in the specific repository
- All submissions are reviewed against the repository spec and acceptance criteria
- Tests are required for all code changes

## Code of conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
