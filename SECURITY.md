# Security Policy

## Reporting a vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Report vulnerabilities using [GitHub private security advisories](https://github.com/Open-Labor-Foundation/open-labor-foundation/security/advisories/new) or by email to **security@openlabor.foundation**.

Include a description of the issue, steps to reproduce, and the potential impact. You can expect acknowledgement within 48 hours and a resolution timeline within 7 days for confirmed vulnerabilities.

## Supported versions

These repositories are in an experimental pre-release state. Security fixes are applied to the current main branch only.

## Scope

- Vulnerabilities in source code in this organization
- Security issues in default configurations
- Hardcoded credentials or secrets

Out of scope: vulnerabilities in third-party dependencies (report upstream), issues requiring physical access.
